## Capstone by Certification Level

### Level 1: Digital Citizen Capstone

**No capstone required** — Complete Track 00 modules and exercises.

### Level 2: Web Developer Apprentice Capstone

**Required After**: HTML + CSS completion

**Project Options:**

| Project | Description | Skills Shown |
|---------|-------------|--------------|
| **Personal Portfolio** | Showcase your work and skills | HTML, CSS, responsive design |
| **Business Landing Page** | Professional page for local business | Layout, typography, images |
| **Restaurant Website** | Menu, location, contact info | Multi-page, forms, styling |
| **Event Page** | Conference or meetup landing page | Design, responsiveness |

**Requirements:**

- Minimum 3 pages
- Mobile responsive
- Contact form (HTML only is fine)
- Professional appearance
- Proper semantic HTML
- Clean, organized CSS

**Example: Local Restaurant Website**

```
Pages Required:
├── index.html (Home - welcome, featured dishes)
├── menu.html (Full menu with prices)
├── about.html (Story, location, hours)
├── contact.html (Contact form, map embed)
└── css/
 └── style.css (All styling)

Features:
• Responsive navigation
• Hero section with background image
• Menu with categories
• Google Maps embed
• Contact form
• Footer with social links
• Works on mobile and desktop
```

### Level 3: JavaScript Developer Capstone

**Required After**: HTML + CSS + JavaScript Basics

**Project Options:**

| Project | Description | Skills Shown |
|---------|-------------|--------------|
| **Interactive Quiz App** | Multiple choice quiz with scoring | DOM, events, logic |
| **Todo List** | Add, complete, delete tasks | CRUD operations, storage |
| **Calculator** | Scientific or finance calculator | Math operations, events |
| **Memory Game** | Card matching game | Event handling, logic |
| **Weather Dashboard** | Show weather data (API) | Fetch, async, JSON |

**Requirements:**

- Full functionality (not just UI)
- Event handling
- DOM manipulation
- User feedback (messages, validation)
- Local storage (if applicable)
- Clean JavaScript code

**Example: Todo List Application**

```
Features Required:
• Add new tasks
• Mark tasks complete/incomplete
• Delete tasks
• Filter (All / Active / Completed)
• Save to localStorage
• Responsive design
• Smooth animations

Code Structure:
├── index.html
├── css/
│ └── style.css
└── js/
 └── app.js
```

### Level 4: Full-Stack Apprentice Capstone

**Required After**: Complete React and Git/GitHub

**Project Options:**

| Project | Description | Skills Shown |
|---------|-------------|--------------|
| **E-commerce Store** | Products, cart, checkout | React, state management |
| **Blog Platform** | Posts, categories, comments | CRUD, routing |
| **Task Management** | Projects, tasks, users | Complex state |
| **Social Dashboard** | Posts, likes, profiles | API integration |

**Requirements:**

- React application
- Multiple components
- Routing (React Router)
- State management
- API integration (can use mock API)
- Authentication (basic)
- Deployed online
- GitHub repository with history

**Example: E-commerce Store**

```
Features Required:
• Product listing with categories
• Product detail pages
• Shopping cart functionality
• Add/remove from cart
• Checkout form
• Order confirmation
• Responsive design
• Search/filter products

Technology:
• React (functional components)
• React Router for navigation
• Context API or Redux for state
• Styled components or CSS modules
• Mock API (json-server or similar)
```

### Level 5: KOOMPI Software Engineer Capstone

**Required After**: Complete all technical tracks + Business track

**Project: Real-World Application**

This is a significant project that:

- Solves a real problem for KOOMPI or a client
- Is deployed and used by real users
- Demonstrates full-stack capabilities
- Shows business understanding
- Can be maintained and scaled

**Options:**

1. **KOOMPI Internal Tool** — Tool that helps the team
2. **Client Project** — Real project for a client
3. **Open Source Contribution** — Significant contribution to an open source project
4. **Startup MVP** — Your own product idea with real users

**Requirements:**

- Complete, production-ready application
- Deployed and accessible
- Documentation (README, API docs if applicable)
- Test coverage (basic)
- Clean code following best practices
- Presentation to team
- Mentor approval

## Capstone Submission Process

### Step 1: Proposal

Before starting, submit a project proposal:

```markdown
# Capstone Project Proposal

## Project Title
[Your project name]

## Problem Statement
[What problem does this solve?]

## Target Users
[Who will use this?]

## Features
- Feature 1
- Feature 2
- Feature 3

## Technology Stack
- [Technologies you'll use]

## Timeline
- Week 1: [Tasks]
- Week 2: [Tasks]
- Week 3: [Tasks]

## How It Demonstrates My Skills
[Connection to what you've learned]
```

### Step 2: Development

- Regular check-ins with mentor
- Use version control (Git)
- Document as you build
- Test throughout development

### Step 3: Submission

Submit:

- Working application (deployed if web-based)
- Source code (GitHub repository)
- README with instructions
- Demo video (2-5 minutes)

### Step 4: Presentation

- Present to mentor and peers
- Explain your decisions
- Demo the functionality
- Answer questions
- Receive feedback

### Step 5: Evaluation

```
EVALUATION CRITERIA:

Technical Implementation (40%)
• Functionality works correctly
• Code quality and organization
• Appropriate use of technologies

Design & User Experience (20%)
• Visual design
• Usability
• Responsive/accessible

Completeness (20%)
• All features working
• Documentation complete
• Deployed (if applicable)

Problem Solving (10%)
• Addresses a real need
• Creative solutions

Presentation (10%)
• Clear explanation
• Answers questions well
```

## Tips for Success

### Do's

 Start with a clear plan
 Build incrementally (MVP first)
 Commit frequently to Git
 Ask for help when stuck
 Test on multiple devices
 Get user feedback
 Document as you go

### Don'ts

 Overscope your project
 Wait until the last minute
 Skip documentation
 Copy code without understanding
 Ignore responsive design
 Submit unfinished work

---

## Getting Started

1. **Choose your project** — Pick something you're excited about
2. **Write proposal** — Get mentor approval
3. **Plan your timeline** — Break into weekly milestones
4. **Build in phases** — MVP → Features → Polish
5. **Test thoroughly** — Try to break your own app
6. **Document everything** — README, comments, commits
7. **Present with pride** — You built something real!

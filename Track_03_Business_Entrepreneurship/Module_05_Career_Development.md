### Goal-Setting Framework

Use **SMART Goals**:

| Letter | Meaning | Bad Goal | Smart Goal |
|--------|---------|----------|------------|
| **S** | Specific | "Get better" | "Learn React" |
| **M** | Measurable | "Know more" | "Build 3 React projects" |
| **A** | Achievable | "Master everything" | "Complete React level" |
| **R** | Relevant | "Learn random stuff" | "Learn what my career needs" |
| **T** | Time-bound | "Someday" | "By June 2025" |

### Your Goal Template

```markdown
## My Career Goal

### 6-Month Goal
- Complete: [Technology/Certification]
- Build: [Project type]
- Achieve: [Metric - WPM, projects, etc.]

### 1-Year Goal
- Position: [Junior Dev, Freelancer, etc.]
- Skills: [List of technologies]
- Income: [Target if applicable]

### 5-Year Vision
- Role: [What you want to be doing]
- Impact: [What difference you'll make]
```

## Lesson 2: Building Your Portfolio

### What is a Portfolio?

Your portfolio is **proof of what you can do**. It shows:

- Your skills
- Your projects
- Your growth
- Your personality

### What to Include

```
PORTFOLIO ESSENTIALS:

1. ABOUT YOU
   - Brief introduction
   - Your story
   - What you're passionate about

2. SKILLS
   - Technologies you know
   - Soft skills
   - Certifications

3. PROJECTS (3-5 quality ones)
   - What it is
   - What problem it solves
   - Technologies used
   - Link to live site
   - Link to code (GitHub)

4. CONTACT
   - Email
   - GitHub
   - LinkedIn (optional)
```

### Project Quality Over Quantity

3 excellent projects > 10 mediocre projects

**What makes a project good:**

- Solves a real problem
- Polished, not half-finished
- Clean code on GitHub
- Has a README explaining it
- Demonstrates your skills
